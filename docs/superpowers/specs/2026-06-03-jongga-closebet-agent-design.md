## 1. 목표와 비목표

**목표**
- 하루 1회 **시황·수급·재료·테마·차트**를 분석해 **유동성 두꺼운 대장주 중심**으로 종가에 살 바스켓을 선정한다.
- 종가에 자동 매수 → **익일 시초가에 (귀속분) 전량 자동 매도**한다 (오버나잇 갭 수익).
- 매 회차 **근거 리포트**를 남겨 복기 자산으로 축적한다.
- "LLM은 분석·선정만, 주문 방아쇠와 리스크 한도는 룰"이라는 분리로 자동매매를 백테스트 가능하고 가드레일이 견고하게 만든다.
- **백테스트 입력 = 라이브 입력(동일 정보집합).** 의사결정 시점에 가용한 정보로만 결정·검증한다(train-serve parity).

**비목표 (YAGNI)**
- **시가배팅·배수배팅·갭다운 홀딩** — Phase 2 별도 모듈. (단, **갭다운/악재 진입 회피**는 MVP RiskGovernor 범위에 포함.)
- 장중 틱/호가 실시간 엔진, 멀티 전략 프레임워크.
- 딥러닝 모델 훈련 — 차트 정량화 수식은 **결정론적 퀀트 팩터**로만 사용.
- 분산 인프라(메시지버스/상시 데몬) — 하루 2-패스 배치로 충분.

## 2. 핵심 결정사항

| # | 결정 | 선택 | 이유 |
|---|---|---|---|
| 1 | 매매법 | **종가베팅 단일** (시가배팅은 Phase 2) | 사용자가 한 가지 매매법으로 시작 요청. 깨끗한 일봉 백테스트 유지 |
| 2 | 선정 두뇌 | **하이브리드** (정량 룰 스크리너 + LLM 재료/시황) | 룰=백테스트 가능 + LLM=재료·테마 맥락 판단 |
| 3 | 자율성 | **완전 자동 실행 + 근거 리포트 동시 생성** | 무인 매수/매도, 리포트는 투명성·복기 자산 |
| 4 | 재료 소스 | **뉴스피드 + LLM 웹검색 병행** | 뉴스(네이버/연합/DART) + 웹검색 보강 |
| 5 | 검증 경로 | **일봉 백테스트 먼저 → 소액 실거래 포워드** | 종가베팅은 일봉 백테스트가 실제로 가능 |
| 6 | 코드 재사용 | **KIS 어댑터(인증·REST·주문)만 재사용, 새 패키지** | 틱/호가 수집기·스캘핑 엔진은 불필요 |
| 7 | 출구 규칙 | **익일 시초 전량매도 — 단 청산 대상=DailyPlan 귀속분, 전용계좌 전제** | 결정론·오버나잇 최소화 + 무관 보유분 오청산 방지(§14 D) |
| 8 | 오케스트레이션 | **A: 결정론적 일일 파이프라인 + 경계지어진 LLM 단계** | daily cadence엔 2-두뇌/동시루프 과잉 |
| 9 | Universe | **유동성(거래대금·호가수용력) 중심, 대형주는 자연 수렴** | 만쥬 1순위=호가두께. 시총 하드컷은 중소형 대장주 선제배제 위험(§14 E) |
| 10 | LLM | **OpenAI GPT-5.4** (OpenAI SDK) | 사용자 지정. structured output + 웹검색 + 프롬프트 캐싱 |
| 11 | 백테스트 데이터 | **pykrx(KRX EOD)**, 라이브=KIS, **데이터 어댑터 추상화** | 수급·시총·상폐 역사는 KIS 부실 → KRX 필요 |
| 12 | **시점 정합성** | **결정·매수 = 15:2x 동시호가 예상체결 스냅샷 / 수급 = t-1 확정** | 14:00 결정 vs t-종가 백테스트의 train-serve skew 제거(§14 A) |
| 13 | **검증 게이트** | **사전등록 합격선 + walk-forward·홀드아웃 + 파라미터 탐색범위** | 합격선 없는 게이트는 과최적화 절차로 변질(§14 C) |
| 14 | **체결모델** | **동시호가 자기주문 시장충격 함수(매수·매도 대칭, 갭다운 가중)** | "단일가=무충격" 가정은 갭수익 체계적 과대평가(§14 B) |

## 3. 아키텍처 & 컴포넌트

하루 **2-패스** 배치. 상시 데몬 아님.

```
══ Pass 1 — 장중 (≈14:00): 사전 리서치 (광의 후보 + 무거운 LLM) ═══════
 1. MarketRegime     지수·거래대금(가용분) → 시황등급 + 비중배수            [룰]
 2. PreScreener      t-1 확정 데이터로 광의 후보군(예 100) 압축              [룰]
 3. CatalystResearch 후보별 뉴스수집 + LLM 웹검색 → 재료·테마(스냅샷 영속화) [LLM+도구]

══ Pass 2 — 동시호가 직전 (≈15:20~15:28): 확정·실행 ════════════════════
 4. FinalScreener    예상체결 스냅샷으로 차트·거래대금 팩터 재계산·랭크       [룰]
 5. Selector         시황+후보(팩터)+재료 → 바스켓·확신도·근거 (룰 fallback) [LLM/룰]
 6. RiskGovernor     한도·호가수용cap·악재공시·블랙리스트 심사                [룰, 거부권]
 7. CloseExecutor    승인 바스켓을 종가 동시호가 매수 (15:28 발주 데드라인)    [KIS 주문]
 8. ReportWriter     시황·후보·재료·선정근거·갭다운시나리오·체결 기록         [리포트]

══ 익일 시가 (≈09:00) ═══════════════════════════════════════════════
 9. OpenExecutor     실잔고 ∩ Plan귀속분 시초 전량매도 + 실현손익 기록        [KIS 주문]
10. (다음 회차 1·3·5에 직전 복기 컨텍스트 제공)
```

> **후보군 일관성:** Pass 1 후보군(t-1 기준 광의, 예 100)은 넉넉히 잡아 Pass 2 최종 후보가 그 **부분집합**이 되게 한다. 15:2x에 새로 진입한 종목은 보강 조사(경량 LLM) 후 편입.

### 핵심 데이터 계약 — `DailyPlan`
분석 단계가 쓰고 실행 단계가 읽는 단 하나의 계약. **각 필드의 생산 단계를 명시**한다(C1·C2 해소).

```
DailyPlan
 ├ date, as_of_decision(=15:2x 스냅샷 시각), as_of_supply(=t-1)
 ├ regime: { 등급(BULL/NEUTRAL/BEAR/HALT), 비중배수 }           # 1 생산
 ├ basket[]: {
 │    symbol,
 │    factors(거래대금·수급·정배열·신고가·거래량비·지지),         # 4 생산(수급은 t-1)
 │    catalyst(요약·강도·신선도·persistence·출처), theme,         # 3 생산
 │    conviction(0~1), rationale, gap_down_scenario,             # 5 생산(룰 fallback시 팩터점수)
 │    weight, target_qty,                                        # 6(Sizer) 생산
 │    filled_qty, avg_price }                                    # 7(체결) 갱신
 ├ risk_limits: { 종목당상한, 총익스포저, 호가수용cap, 블랙리스트, 악재플래그 }
 ├ attribution: { intended_symbols, intended_qty }              # 귀속(익일 청산 범위 한정용)
 └ report_path
```

- **영속화:** SQLite. 복기 + 크래시 복구. CatalystResearch 웹검색 결과도 **스냅샷 영속화**(시점통제·복기, §14 DF-5).
- **정합성 원칙:** 익일 매도(9)는 `DailyPlan`을 맹신하지 않되, **청산 범위는 `실잔고 ∩ attribution`** 으로 한정. Plan은 "의도", 잔고는 "실제", 귀속은 "교집합".

### 컴포넌트 분리 원칙
- 분석·실행은 `DailyPlan` 하나로만 통신 → 독립 이해/테스트.
- `RiskGovernor`는 실행과 분리된 게이트 → 선정이 뭘 하든 한도 강제.
- **데이터 소스 어댑터**(`MarketDataProvider`)가 KRX(pykrx)/KIS에 묶이는 유일한 곳. 단, 두 소스가 **같은 의미의 수치**를 내도록 정규 스키마 + 골든 테스트로 보장(§8.3).

## 4. 데이터 흐름 (하루 사이클)

```
══ Pass 1 (≈14:00) ══
 ① MarketRegime: 지수·거래대금(가용분) → 시황등급 + 비중배수
 ② PreScreener: t-1 확정 데이터로 광의 후보군(예 100) 압축
 ③ CatalystResearch(LLM): 후보별 뉴스 + 웹검색 → 재료/테마 구조화(스냅샷 저장)

══ Pass 2 (≈15:20~15:28, 동시호가 직전) ══
 ④ FinalScreener: 예상체결 스냅샷으로 차트·거래대금 팩터 재계산 → 랭크
 ⑤ Selector: 시황+후보(팩터)+재료 → 바스켓 K개 + 확신도 + 갭다운 시나리오
 ⑥ RiskGovernor: 한도·호가수용·악재공시·블랙리스트 심사 → 승인 바스켓
 ⑦ CloseExecutor: 종가 동시호가 매수(15:28 데드라인) → 실체결 DailyPlan 갱신
 ⑧ ReportWriter: 전 과정 리포트(MD)

══ 익일 시가 (≈09:00) ══
 ⑨ OpenExecutor: 실잔고 ∩ Plan귀속분 → 시초가 전량매도 → 실현손익 → 복기
```

### 4가지 핵심 규칙
1. **방아쇠는 항상 룰.** LLM은 ③⑤(재료조사·선정)만. 매수/매도 실행·한도는 전부 룰.
2. **결정 시점 = 가용 정보만.** 결정·매수는 15:2x 예상체결 스냅샷, 수급은 t-1 확정치. 미래(확정 종가·당일 수급) 참조 금지 → 백테스트=라이브.
3. **출구는 귀속분만 무조건.** 익일 시초 전량매도는 `실잔고 ∩ attribution`에 한정, 손익·갭 무관(MVP). 갭다운 시나리오는 기록만(진입 회피는 ⑥에서).
4. **잔고가 진실.** 실행은 KIS 실잔고에서 동기화하고 시작 — 내부 상태 맹신 금지.

## 5. 종목 선정 상세

### 5.1 Universe (모집단) — 유동성 중심
만쥬 1순위 기준은 **호가창 두께**다("시총 커도 호가 얇으면 탈락, 시총 낮아도 두꺼우면 매매").

- **1차 하드필터(되돌릴 수 없는 제거는 최소):** ① 거래대금 상위 K ∩ ② 호가수용력(슬리피지 없이 목표 비중 소화 가능) ∩ ③ 거래정지·관리·정리매매 제외 ∩ ④ 동전주·상폐임박 배제 수준의 **매우 낮은 안전 시총 하한**.
- **대형주 지향은 하드컷이 아니라 자연 수렴:** "대규모 자금은 대장주 안에서만"(사용자 종가베팅 지시)은 **거래대금·호가수용 cap × 비중**으로 자동 충족 → 거래대금 충분한 중소형 테마 대장주를 선제 배제하지 않음.
- **시총은 랭킹 팩터**로 사용(하드 AND 아님). 컷은 오픈 파라미터(§12.1).

### 5.2 Stage 4 — 스크리너 팩터 (전부 결정론적)
**as-of 시각(§14 A):** 차트·거래대금 팩터의 `Price_t`/거래대금 = **15:2x 동시호가 예상체결가/누적값**. 수급은 **t-1 확정 + 최근 5일 누적**(당일 수급은 장 마감 후 집계라 결정 시점 미가용). **데이터 누수 방지:** 롤링 통계는 결정일 가용분까지만. **정상성:** 절대가격 금지, 비율·거리·로그수익률만.

**① 거래대금** — 절대값 + 시장 내 상대순위(0~1). (Pass2 예상체결 누적)
**② 수급** — 외국인·기관 순매수(**t-1 확정** + 최근 5일 누적). 정규화 = 순매수금액 / 거래대금. 카테고리 정의·소스 매핑은 §8.3 정규 스키마.
**③ 차트 팩터:**

```
정배열  Spread_t      = (MA_short − MA_long) / MA_long           # >0이면 단기선 위, 클수록 강함
        Alignment     = 1.0  if MA5>MA20>MA60>MA120
                      = (충족 쌍 수 / 3) 로 연속 보간            # 완전 역배열 → 0
신고가  Proximity_N   = Price_t / max(Price_{t..t-N+1})          # =1.0이면 N일 신고가
        DaysSinceHigh = t − argmax(Price_{t..t-N+1})            # 0이면 오늘 신고가(박스권 돌파)
거래량  VolRatio      = mean(양봉일 거래량) / mean(음봉일 거래량) # >1이면 상승매집 흔적
지지    NearMA        = 현재가가 MA5/MA20 ±x% 이내 + 직전 반등   # 눌림목 proxy(초기 단순형, 후속 정교화)
```

**랭킹:** 하드필터 통과분에 팩터 가중합 점수 → **상위 N개**. 가중치·룩백 N은 오픈 파라미터(§12.1, 탐색범위·합격선과 함께 확정).

### 5.3 Stage 3 — CatalystResearch (LLM + 도구)
후보별로 뉴스피드(네이버/연합·DART 공시) 수집 + **GPT-5.4 웹검색** → 구조화 산출:
`{ has_catalyst, summary, strength, recency, persistence, theme, sources[] }`.
- **`persistence`(§14 TL-6):** 재료가 *이미 종가에 반영됐는지 vs 익일까지 끌고 갈 미반영 잔존 모멘텀인지* 구분 — recency만으로는 "늦은 신호"를 못 거름.
- 검색 결과는 **스냅샷 영속화**(시점통제·복기). 시스템 프롬프트는 프롬프트 캐싱.

### 5.4 Stage 5 — Selector (LLM, 스키마 강제 구조화 출력)
- 입력: 시황등급 + 후보(팩터) + 재료조사.
- 출력: 랭크된 바스켓 `{ symbol, conviction(0~1), rationale, theme, gap_down_scenario }[]`.
- GPT-5.4가 거래대금·수급·차트·재료·테마를 종합해 최종 K개 + 확신도. 진입 전 **갭다운 대응 시나리오**도 종목별 기록(실행은 ⑥ 진입 회피 / 홀딩은 Phase 2).

### 5.5 룰 baseline Fallback Selector (§14 C1·C2 — 백테스트 구현 가능성)
백테스트의 **룰 baseline은 LLM(3·5)이 꺼진다.** 이때 결정론적 대체:
- **선정:** §5.2 팩터 가중합 점수 상위 K.
- **conviction:** 정규화 팩터점수(0~1).
- **weight:** 팩터점수 비례(또는 균등 — 둘 다 백테스트 비교).
- catalyst/theme 필드는 baseline에서 공란(또는 룰 테마태그). → baseline은 "재료 없이 정량 팩터만의 엣지"를 측정.

## 6. Sizing & 리스크 거버너

```
종목 비중 = 자본 × 시황배수 × conviction   (바스켓 내 정규화)
   cap by  종목당상한 · 총익스포저상한 · 호가수용cap
```

- **시황배수:** BULL 1.0 / NEUTRAL 0.5 / BEAR 0.2 / HALT 0. + "파격 비중"용 **강세확신 tier(예 STRONG_BULL 1.5)** 옵션 — 단 항상 호가수용cap에 종속.
- **호가수용 cap(§14 B):** 분모를 **"동시호가 추정 체결량 대비 비중"** 으로 정의(일거래대금 x%가 아님 — 동시호가 거래량은 일거래대금의 일부). 실행 직전 **KIS 예상체결가/수량(예상체결 TR)** 으로 재확인.
- **거버너 거부권(리스크 늘리는 행동만 거부):**
  - HALT → 매수 0.
  - 종목당상한·총익스포저 초과 → 감액/제외.
  - **당일 악재성 공시 감지(유증·감자·블록딜·관리지정 등) → 진입 거부(§14 ER-3).**
  - 블랙리스트(거래정지·최근 연속손실) → 제외.
- **매도는 막지 않음** — 익일 시초 청산은 항상 통과(가속).

## 7. 실행 & 정합성

### 7.1 Stage 7 — 종가 매수
- **종가 동시호가(15:20~15:30) 매수**, **15:28 발주 데드라인 가드**(§14 ER-6). 예상체결가 기반 지정가 톨러런스는 오픈 파라미터.
- 체결조회로 실수량 확인 → `basket[].filled_qty/avg_price` 및 `attribution` 갱신.

### 7.2 Stage 9 — 시초 매도
- 익일 **KIS 실보유 잔고 조회 → `실잔고 ∩ attribution` 만 시초가 전량매도**(시가 동시호가/시가 시장가). 실현손익 기록.
- **전용 계좌 전제**(에이전트 외 보유분과 분리). 잔고에 있으나 attribution에 없는 종목 → **청산하지 않고 알림**(§14 ER-5).
- 시초 **하한가·거래정지로 매도 불가** 시 → 분기 처리(미체결 추적·재시도·알림, §14 ER-4). 갭다운 시나리오는 기록만, 그 외엔 무조건 청산.

### 7.3 에러 / 정합성 (실거래 필수)
| 상황 | 대응 |
|---|---|
| 부분/미체결 | 체결조회로 실수량 확인 → 익일 매도는 실잔고 기준 |
| **청산 귀속(ER-5)** | 청산 = `실잔고 ∩ attribution`. 무관 보유분 보호. 잔고>Plan 종목은 알림 후 보류 |
| **시초 청산 불가(ER-4)** | 하한가/거래정지로 미체결 → 추적·재시도·강알림 |
| 크래시·재시작 | 내부 상태 맹신 금지, KIS 실잔고 동기화로 시작 |
| 중복 주문 | 주문ID 멱등성 + 당일 1회 실행 가드 |
| **15:28 데드라인(ER-6)** | 발주 마감 가드 — 동시호가 마감 직전 미발주분 처리 정책 |
| 14:00/15:2x 누락(머신 꺼짐) | 그날 진입 skip(반쪽 진입 방지) |
| 09:00 누락 | **강한 알림**(오버나잇 방치 위험) + 수동 청산 경로 |
| 사람 개입 | Kill switch — 수동 전량 청산·중단 |
| API 오류 | 재시도 + 실패 시 알림, 해당 종목만 skip(전체 중단 X) |

## 8. 백테스트 & 검증

### 8.1 두 층 분리
| 층 | 단계 | 백테스트 |
|---|---|---|
| 룰 | 1·2·4·6·7·9 | **일봉으로 결정론적 재현 가능** (룰 baseline = §5.5 fallback) |
| LLM | 3·5 | 과거 뉴스 PIT 재현·비결정 → 백테스트에서 끄고 룰 baseline, 효과는 포워드로 측정 |

### 8.2 엔진 (as-of 강제 + 시장충격 체결모델)
- 매 거래일 `종가매수 → 익일시초매도` 시뮬.
- **as-of 강제(§14 A):** 수급 = t-1 확정, 차트·거래대금 팩터 = 결정 시점 가용가(백테스트는 t 종가를 15:2x 예상체결의 근사로 사용 — **이 근사 갭을 §8.4 민감도에 명시**). 매수 t 종가, 매도 t+1 시초.
- **체결모델(§14 B):** "무충격" 폐기. **주문수량 대비 동시호가 추정 가용물량 기반 시장충격/슬리피지 함수**(매수·매도 **대칭**, 갭다운 구간 슬리피지 가중). cap 분모 = 동시호가 추정 체결량.
- **상폐/거래정지로 시초 청산 불가했던 케이스의 청산가 분기(§14 BT-5)** 포함 — 없으면 손실 꼬리 누락(낙관 편향).
- **누수 방지:** 팩터는 결정 시점 가용분까지만, 미래값 금지.
- 메트릭: 승률·평균갭수익·MDD·회전율, **net**(수수료 + 거래세 0.18~0.23% + 슬리피지 차감).

### 8.3 데이터 소스 (어댑터 추상화 + 정규 스키마)
- **백테스트 = pykrx(KRX EOD):** 일봉 OHLCV, 투자자별 수급, 시총·상장주식수, 상장/상폐 목록(생존편향 제거). 무료. 비공식·레이트리밋·PIT 한계(§14 DF-4) 주의.
- **라이브 = KIS REST:** 일봉·순위·수급·예상체결·현재가·주문·잔고.
- **정규 수급 스키마(§14 DF-2):** "외국인/기관"의 세부 카테고리(연기금·사모·투신·기타법인 포함 여부)를 정규 스키마로 고정하고 **pykrx↔KIS 매핑을 골든 테스트로 검증** → 두 소스가 같은 종목에 같은 수급값. 어댑터 완료 게이트(Phase 1).

### 8.4 검증 게이트 — 프로토콜·합격선 사전 등록 (§14 C)
- **OOS 프로토콜:** walk-forward(롤링 재최적화) + 최종 홀드아웃(미사용 기간) 검증.
- **사전 등록 합격선(파라미터 탐색 전 확정):** net 기대갭수익 > 비용, MDD 한도, 최소 표본수·승률, **팩터↔1박갭 정보계수(IC) 유의성**(§14 TL-1 직접 해소).
- **파라미터 탐색범위(§12.1)와 다중검정 주의:** "통과할 때까지 맞추기" 방지 — 탐색은 walk-forward 내부에서만, 홀드아웃은 1회만 본다.

### 8.5 정직성
백테스트 = "기계적 엣지 거르는 1차 필터". LLM 선정 품질은 **소액 실거래 포워드 + 복기**로 최종 검증.
