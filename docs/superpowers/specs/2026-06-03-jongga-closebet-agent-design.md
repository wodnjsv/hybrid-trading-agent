## 1. 목표와 비목표

**목표**
- 하루 1회 **시황·수급·재료·테마·차트**를 분석해 **유동성 두꺼운 대장주 중심**으로 종가에 살 바스켓을 선정한다.
- 종가에 자동 매수 → **익일 시초가에 (귀속분) 전량 자동 매도**한다 (오버나잇 갭 수익).
- 매 회차 **근거 리포트**를 남겨 복기 자산으로 축적한다.
- "LLM은 분석·선정만, 주문 방아쇠와 리스크 한도는 룰"이라는 분리로 가드레일을 견고하게 한다.

**핵심 설계 원칙 (수정 — §14 1·4)**
- **선정 신호는 'as-of 안전(t-1 확정)' 팩터를 기준선으로 한다.** 백테스트는 라이브를 *근사*하되, **동시호가 마지막 수분의 가격 드리프트는 EOD 데이터로 복원 불가능한 잔여 look-ahead**다 — "백테스트 입력 = 라이브 입력"은 *완전 달성 불가*이며, 이 잔여를 §8.4에서 명시적으로 측정·관리한다(은폐 금지).
- **검증의 비대칭을 인정한다.** historical 백테스트는 "기계적 바닥 + 정량 팩터 엣지"만 검증한다. 가설상 진짜 알파인 **LLM 재료/테마 선정층은 historical 백테스트 불가** → 포워드 A/B로만 검증되며, 이것이 자본 증액의 게이트다(§8.5).

**비목표 (YAGNI)**
- **시가배팅·배수배팅·갭다운 홀딩** — Phase 2 별도 모듈. (단, 마감 전 공개된 악재 **진입 회피**와 장전 **재점검 알림**은 MVP 범위.)
- 장중 틱/호가 실시간 엔진, 멀티 전략 프레임워크.
- 딥러닝 모델 훈련 — 차트 정량화 수식은 **결정론적 퀀트 팩터**로만 사용.
- 분산 인프라(메시지버스/상시 데몬) — 배치로 충분.

## 3. 아키텍처 & 컴포넌트

**3-시점 배치.** 상시 데몬 아님.

```
══ Pass 1 — 장중 (≈14:00): 사전 리서치 (광의 후보 + 무거운 LLM) ═══════
 1. MarketRegime     t-1 확정 지수·시장거래대금 → 시황등급 + 비중배수         [룰]
 2. PreScreener      t-1 확정 데이터로 광의 후보군(예 100~150) 압축           [룰]
 3. CatalystResearch 후보별 뉴스수집 + LLM 웹검색 → 재료·테마(스냅샷 영속화)  [LLM+도구]

══ Pass 2 — 동시호가 직전 (≈15:20~15:28): 확정·실행 ════════════════════
 4. FinalScreener    t-1 안전팩터 랭크 + 15:2x 예상체결 veto(급락/거래정지)    [룰]
 5. Selector         시황+후보(팩터)+재료 → 바스켓·근거·conviction(랭킹용)     [LLM/룰 fallback]
 6. RiskGovernor     정량가중 sizing + 한도·호가수용cap·악재·블랙리스트 심사     [룰, 거부권]
 7. CloseExecutor    승인 바스켓을 종가 동시호가 매수 (15:28 발주 데드라인)     [KIS 주문]
 8. ReportWriter     시황·후보·재료·선정근거·갭다운시나리오·체결 기록          [리포트]

══ 익일 장전 (≈08:30~09:00): 재점검 ════════════════════════════════════
 9. PreOpenCheck     귀속분의 거래정지/하한가/오버나잇 악재 점검 → 강알림      [룰+알림]

══ 익일 시가 (≈09:00) ═════════════════════════════════════════════════
10. OpenExecutor     실잔고 ∩ 귀속분 시초 전량매도 + 실현손익 기록            [KIS 주문]
11. (다음 회차 1·3·5에 직전 복기 컨텍스트 제공)
```

> **후보군 일관성(§14 2):** Pass 2 최종 후보는 Pass 1 후보군의 **strict 부분집합**이다. PreScreener는 t-1 거래대금 기준 **광의(넓은 그물, 예 100~150)**로 잡아 누락을 줄인다. **14:00~15:2x 사이 장중 새로 부상한 종목은 MVP 범위 밖(로그만)** — 대형주 중심이라 영향이 제한적이고, t-1 거래대금 lag 편향은 §14 2의 알려진 한계로 둔다(Phase 2에서 장중 부상 탐지 보강).

### 핵심 데이터 계약 — `DailyPlan`
분석 단계가 쓰고 실행 단계가 읽는 단 하나의 계약. **각 필드의 생산 단계를 명시**한다.

```
DailyPlan
 ├ date, as_of_signal(=t-1), as_of_exec(=15:2x 예상체결 스냅샷 시각)
 ├ regime: { 등급(BULL/NEUTRAL/BEAR/HALT), 비중배수 }              # 1 생산(t-1)
 ├ basket[]: {
 │    symbol,
 │    factors(거래대금·수급·정배열·신고가·거래량비·지지),            # 4 생산(전부 t-1 안전)
 │    catalyst(요약·강도·신선도·persistence·출처), theme,            # 3 생산
 │    rank, conviction(0~1, 랭킹·기록용 — 사이징 미사용),            # 5 생산(룰 fallback=팩터점수)
 │    rationale, gap_down_scenario,                                  # 5 생산
 │    weight, target_qty,                                           # 6(Sizer, 정량가중) 생산
 │    filled_qty, avg_price }                                       # 7(체결) 갱신
 ├ risk_limits: { 종목당상한, 총익스포저, 호가수용cap, 블랙리스트, 악재플래그 }
 ├ attribution: { intended_symbols, intended_qty }                 # 익일 청산 범위 한정용
 └ report_path
```

- **영속화:** SQLite. CatalystResearch 웹검색 결과도 **스냅샷 영속화**(시점통제·복기).
- **정합성 원칙:** 익일 매도(10)의 **청산 범위 = `실잔고 ∩ attribution`**. Plan=의도, 잔고=실제, 귀속=교집합.

### 컴포넌트 분리 원칙
- 분석·실행은 `DailyPlan` 하나로만 통신.
- `RiskGovernor`는 실행과 분리된 게이트(거부권).
- **데이터 어댑터(`MarketDataProvider`)**가 KRX(pykrx)/KIS에 묶이는 유일한 곳. 두 소스가 **같은 의미의 수치**를 내도록 정규 스키마 + 골든 테스트로 보장(§8.3).

## 4. 데이터 흐름 (하루 사이클)

```
══ Pass 1 (≈14:00) ══  ① 시황등급(t-1) → ② 광의 후보(t-1) → ③ LLM 재료조사(스냅샷)
══ Pass 2 (≈15:20~15:28) ══  ④ t-1 안전팩터 랭크 + 예상체결 veto → ⑤ 바스켓 선정
                              → ⑥ 정량가중 sizing + 심사 → ⑦ 종가 동시호가 매수 → ⑧ 리포트
══ 익일 장전 (≈08:30~09:00) ══  ⑨ PreOpenCheck(거래정지·하한가·악재 알림)
══ 익일 시가 (≈09:00) ══  ⑩ 실잔고∩귀속분 시초 전량매도 → 손익 → 복기
```

### 4가지 핵심 규칙
1. **방아쇠는 항상 룰.** LLM은 ③⑤(재료조사·선정)만. 실행·한도·사이징은 전부 룰.
2. **결정 시점 = 가용 정보만.** 선정 팩터·수급 = t-1 확정. 15:2x 예상체결은 *실행가·veto*에만. 확정 종가(미래값) 참조 금지 — 잔여 드리프트는 §8.4 측정.
3. **출구는 귀속분만 무조건.** 시초 전량매도는 `실잔고 ∩ attribution`에 한정, 손익·갭 무관(MVP).
4. **잔고가 진실.** 실행은 KIS 실잔고에서 동기화하고 시작.

## 5. 종목 선정 상세

### 5.1 Universe (모집단) — 유동성 중심
만쥬 1순위 기준은 **호가창 두께**다.

- **1차 하드필터(되돌릴 수 없는 제거는 최소):** ① 거래대금 상위 K ∩ ② 호가수용력(목표 비중 슬리피지 없이 소화) ∩ ③ 거래정지·관리·정리매매 제외 ∩ ④ 동전주·상폐임박 배제 수준의 **매우 낮은 안전 시총 하한**.
- **대형주 지향은 하드컷이 아니라 자연 수렴:** 거래대금·호가수용 cap × 비중으로 자동 충족 → 거래대금 충분한 중소형 테마 대장주를 선제 배제하지 않음.
- **시총은 랭킹 팩터**(하드 AND 아님). 컷은 §12.1.

### 5.2 Stage 4 — 스크리너 팩터 (전부 결정론적, **t-1 안전**)
**as-of 시각(§14 1):** 선정 팩터는 전부 **t-1 확정 데이터**로 계산(차트 MA/정배열/신고가/거래량비, 수급 모두). 오늘(t)의 강도에 의존하는 신호는 백테스트 불가·낙관편향원이므로 **선정에 쓰지 않고**, 15:2x 예상체결은 **veto(급락·거래정지·악재 시 제외)와 실행가**에만 쓴다. **누수 방지:** 롤링 통계는 t-1까지만. **정상성:** 비율·거리·로그수익률만.

**① 거래대금** — t-1 절대값 + 시장 상대순위(0~1).
**② 수급** — 외국인·기관 순매수(t-1 확정 + 최근 5일 누적). 정규화 분모 = **동일 t-1 윈도우 거래대금**(§14 m3). 카테고리 정의·소스 매핑은 §8.3 정규 스키마.
**③ 차트 팩터(t-1 종가 기준):**

```
정배열  Spread_t      = (MA_short − MA_long) / MA_long
        Alignment     = 1.0 if MA5>MA20>MA60>MA120 ; else (충족 쌍 수 / 3) 보간
신고가  Proximity_N   = Price_{t-1} / max(Price_{t-1..t-N})       # =1.0이면 t-1이 N일 신고가
        DaysSinceHigh = (t-1) − argmax(Price_{t-1..t-N})
거래량  VolRatio      = mean(양봉일 거래량)/mean(음봉일 거래량)
지지    NearMA        = Price_{t-1}가 MA5/MA20 ±x% 이내 + 직전 반등
```

**랭킹:** 하드필터 통과분에 팩터 가중합 → 상위 N개. 가중치·룩백 N은 §12.1(탐색범위·합격선과 함께 사전 등록).

### 5.3 Stage 3 — CatalystResearch (LLM + 도구)
후보별 뉴스(네이버/연합·DART) + GPT-5.4 웹검색 → `{ has_catalyst, summary, strength, recency, persistence, theme, sources[] }`.
- **`persistence`(§14 TL-6):** 재료가 *이미 가격 반영* vs *익일까지 끌 미반영 잔존 모멘텀*인지 구분.
- 검색 결과 **스냅샷 영속화**(시점통제·복기). 시스템 프롬프트 캐싱.

### 5.4 Stage 5 — Selector (LLM, 구조화 출력)
- 입력: 시황등급 + 후보(t-1 팩터) + 재료조사. 출력: 랭크 바스켓 `{ symbol, conviction, rationale, theme, gap_down_scenario }`.
- conviction은 **선정·랭킹·리포트용**이며 **사이징 승수로 쓰지 않는다**(§14 5, 미보정). 갭다운 시나리오는 기록 + ⑥ 진입 회피용(홀딩은 Phase 2).

### 5.5 룰 baseline Fallback Selector (§14 C1·C2)
백테스트 룰 baseline은 LLM(3·5)이 꺼진다:
- 선정 = §5.2 팩터 가중합 상위 K / conviction = 정규화 팩터점수 / weight = §6 정량가중. catalyst·theme 공란. → "재료 없는 정량 팩터만의 엣지" 측정.

## 6. Sizing & 리스크 거버너

```
종목 비중 = 자본 × 시황배수 × 정량가중(균등 또는 팩터점수 비례)
   cap by  종목당상한 · 총익스포저상한 · 호가수용cap
```

- **사이징은 정량가중만(§14 5).** LLM conviction은 선정·랭킹·기록에만, **사이징 미사용** — 미보정 0~1 숫자를 포지션 크기로 직결하지 않음. 실현 1박갭 대비 보정 데이터가 Phase 4에 쌓이면 conviction-가중 사이징 도입 검토. → **라이브 사이징 경로 = 백테스트 경로.**
- **시황배수:** BULL 1.0 / NEUTRAL 0.5 / BEAR 0.2 / HALT 0. + 강세확신 tier(예 STRONG_BULL 1.5) — 항상 호가수용cap 종속.
- **호가수용 cap(§14 B):** 분모 = **동시호가 추정 체결량 대비 비중**(일거래대금 x% 아님). 실행 직전 **KIS 예상체결가/수량(예상체결 TR)** 재확인.
- **거버너 거부권:** HALT→매수0 / 한도초과→감액·제외 / **마감 전 공개 악재공시(유증·감자·블록딜·관리지정) 감지→진입거부** / 블랙리스트→제외.
  - ⚠️ **악재필터 한계(§14 3):** 이 필터는 *15:28까지 공개된* 악재만 거른다. DART 악재 상당수는 마감 후(16~18시)라 **오버나잇 최대 리스크는 진입필터로 못 막는다.** 그 리스크는 **종목당 사이징·바스켓 분산·익일 시초 청산**으로만 완화되며, 장전 ⑨에서 *감지·알림*한다(MVP는 매도 결정 불변).
- **매도는 막지 않음.**

## 8. 백테스트 & 검증

### 8.1 두 층 분리
| 층 | 단계 | 백테스트 |
|---|---|---|
| 룰 | 1·2·4·6·7·10 | **일봉으로 결정론적 재현**(룰 baseline=§5.5) |
| LLM | 3·5 | historical 재현·비결정 불가 → 포워드 A/B로만(§8.5) |

### 8.2 엔진 (as-of 강제 + 시장충격 + 강제다일 모델)
- 매 거래일 `종가매수 → 익일시초매도` 시뮬.
- **as-of 강제(§14 1):** 선정 팩터·수급 = **t-1 확정**. 매수가 = t 종가(실행). t 종가는 15:2x 예상체결의 근사 — **마지막 수분 드리프트는 EOD 복원 불가한 잔여 한계**로 §8.4 민감도에서 측정(은폐 금지).
- **체결모델(§14 B):** "무충격" 폐기. **주문량 대비 동시호가 추정 가용물량 기반 시장충격/슬리피지 함수**(매수·매도 대칭, 갭다운 가중). cap 분모 = 동시호가 추정 체결량.
- **강제 다일 보유(§14 m1/BT-5):** 시초 하한가/거래정지로 청산 불가한 케이스는 **익일 이후 첫 체결 가능 시점/가격으로 청산하되 손실 종목의 후속 불리 드리프트를 보수적으로 반영**(hand-wave 금지). 손실 꼬리 누락 방지.
- **누수 방지:** 미래값 참조 금지.
- 메트릭: 승률·평균갭수익·MDD·회전율, **net**(수수료 + 거래세 0.18~0.23% + 슬리피지).

### 8.3 데이터 소스 (어댑터 + 정규 스키마)
- **백테스트 = pykrx(KRX EOD):** 일봉·투자자별 수급·시총·상장/상폐(생존편향 제거). 무료. 비공식·레이트리밋·PIT 한계(§14 DF-4) 주의.
- **라이브 = KIS REST:** 일봉·순위·수급·예상체결·현재가·주문·잔고.
- **정규 수급 스키마(§14 DF-2):** "외국인/기관" 세부 카테고리(연기금·사모·투신·기타법인 포함 여부) 정규화 + **pykrx↔KIS 매핑 골든 테스트**. 어댑터 완료 게이트(Phase 1).

### 8.4 검증 게이트 — 사전 등록 (§14 6·1·C)
파라미터 탐색 *전에* 합격선·프로토콜을 등록한다. walk-forward(롤링) + 최종 홀드아웃(1회만).
1. **[1순위·load-bearing] 팩터의 1박 갭 예측력:** 각 팩터의 close→open 수익 정보계수(IC) 유의성. **~0이면 전략 가설 붕괴 → 중단**(§14 6).
2. **기계적 바닥:** 종가→시초 갭이 비용 net으로 유의한 엣지(룰 baseline). MDD·최소 표본·승률 한도.
3. **드리프트 민감도(§14 1):** 선정 팩터를 t-1 vs (참고용) t-종가로 계산해 종목 회전·수익 델타를 측정 → 결과가 막판 드리프트에 의존하면 아티팩트로 간주.
- **다중검정 주의:** 탐색은 walk-forward 내부에서만, 홀드아웃은 1회만 본다.

### 8.5 검증 아키텍처와 그 비대칭 (§14 4 — 핵심 리스크)
- historical 백테스트(§8.4)는 **바닥 + 정량 팩터 엣지**만 검증한다. 가설상 진짜 알파인 **LLM 재료/테마 선정은 historical 백테스트 불가**.
- 따라서 **LLM 증분은 포워드 A/B로만 검증한다:** 같은 포워드 기간에 *LLM 선정 바스켓 vs 룰 baseline 바스켓*을 병렬 기록해 실현 1박갭을 비교. **이 A/B 우위가 자본 증액의 게이트**다 — 룰 baseline 통과만으로 증액하지 않는다.
- 함의(정직): 핵심 thesis는 소액 포워드 전까지 검증되지 않는다. 이 비대칭을 알고 단계(Phase)를 설계한다.
