## 1. 목표와 비목표

**목표**
- 하루 1회 **시황·수급·재료·테마·차트**를 분석해 **대형주/대장주 중심**으로 종가에 살 바스켓을 선정한다.
- 종가에 자동 매수 → **익일 시초가에 전량 자동 매도**한다 (오버나잇 갭 수익).
- 매 회차 **근거 리포트**를 남겨 복기 자산으로 축적한다.
- "LLM은 분석·선정만, 주문 방아쇠와 리스크 한도는 룰"이라는 분리로 자동매매를 백테스트 가능하고 가드레일이 견고하게 만든다.

**비목표 (YAGNI)**
- **시가배팅·배수배팅·갭다운 홀딩** — Phase 2 별도 모듈로 분리 (MVP는 순수 종가베팅, 갭다운 시나리오는 *기록만*).
- 장중 틱/호가 실시간 엔진, 멀티 전략 프레임워크.
- 딥러닝 모델 훈련 — 차트 정량화 수식은 **결정론적 퀀트 팩터**로만 사용.
- 분산 인프라(메시지버스/상시 데몬) — 하루 2회 배치로 충분.

## 2. 핵심 결정사항

| # | 결정 | 선택 | 이유 |
|---|---|---|---|
| 1 | 매매법 | **종가베팅 단일** (시가배팅은 Phase 2) | 사용자가 한 가지 매매법으로 시작 요청. 깨끗한 일봉 백테스트 유지 |
| 2 | 선정 두뇌 | **하이브리드** (정량 룰 스크리너 + LLM 재료/시황) | 룰=백테스트 가능 + LLM=재료·테마 맥락 판단. 만쥬식(재료 질 평가 + 정량 소거) |
| 3 | 자율성 | **완전 자동 실행 + 근거 리포트 동시 생성** | 무인 매수/매도, 리포트는 투명성·복기 자산 |
| 4 | 재료 소스 | **뉴스피드 + LLM 웹검색 병행** | 재료 해석의 핵심. 뉴스(네이버/연합/DART) + 웹검색 보강 |
| 5 | 검증 경로 | **일봉 백테스트 먼저 → 소액 실거래 포워드** | 종가베팅은 스캘핑과 달리 일봉 백테스트가 실제로 가능 |
| 6 | 코드 재사용 | **KIS 어댑터(인증·REST·주문)만 재사용, 새 패키지** | 틱/호가 수집기·스캘핑 엔진은 종가베팅에 불필요 |
| 7 | 출구 규칙 | **익일 시초가 무조건 전량매도** | 결정론적·백테스트 명확·오버나잇 노출 최소 |
| 8 | 오케스트레이션 | **A: 결정론적 일일 파이프라인 + 경계지어진 LLM 단계** | daily cadence엔 2-두뇌/동시루프 과잉. LLM은 단계 내로 제한 |
| 9 | Universe | **대형주·대장주 중심** (잡주 제외) | 만쥬: 대규모 자금은 거래대금 빨아들이는 대장주 안에서만 |
| 10 | LLM | **OpenAI GPT-5.4** (OpenAI SDK) | 사용자 지정. structured output + 웹검색 도구 + 프롬프트 캐싱 |
| 11 | 백테스트 데이터 | **pykrx(KRX EOD)**, 라이브=KIS, **데이터 어댑터 추상화** | 수급·시총·상폐 역사는 KIS 부실 → KRX 필요. 무료로 시작 |

## 3. 아키텍처 & 컴포넌트

하루 2회 트리거되는 **결정론적 파이프라인**. 상시 데몬 아님.

```
══ 장 마감 전 (≈14:00 트리거) ═══════════════════════════
 1. MarketRegime     지수·시장거래대금 → 시황등급 + 비중배수          [룰]
 2. UniverseScreener 대형주 모집단 → 거래대금·수급·차트팩터 필터·랭크   [룰]
 3. CatalystResearch 후보별 뉴스수집 + LLM 웹검색 → 재료·테마 정리     [LLM+도구]
 4. Selector         시황+후보+재료 종합 → 바스켓·확신도·근거 결정       [LLM, 구조화출력]
 5. RiskGovernor     비중한도·종목당상한·호가수용cap·블랙리스트 심사     [룰, 거부권]
 6. CloseExecutor    승인 바스켓을 종가에 매수                         [KIS 주문]
 7. ReportWriter     시황·후보·재료·선정근거·갭다운시나리오·체결 기록    [리포트]

══ 익일 시가 (≈09:00 트리거) ════════════════════════════
 8. OpenExecutor     KIS 실보유 잔고 조회 → 시초가 전량매도 + 손익 기록  [KIS 주문]
 9. (다음 회차 1·3·4에 직전 복기 컨텍스트 제공)
```

### 핵심 데이터 계약 — `DailyPlan`
분석 단계(1~4)가 쓰고 실행 단계(6,8)가 읽는 단 하나의 계약:

```
DailyPlan
 ├ date, regime: { 등급(BULL/NEUTRAL/BEAR/HALT), 비중배수 }
 ├ basket[]: { symbol, conviction(0~1), weight, target_qty, rationale,
 │             factors(거래대금·수급·정배열·신고가·거래량비·지지),
 │             catalyst(요약·강도·신선도·출처), theme, gap_down_scenario(기록용),
 │             filled_qty, avg_price }                     # 6에서 실체결 갱신
 ├ risk_limits: { 종목당상한, 총익스포저, 호가수용cap, 블랙리스트 }
 └ report_path
```

- **영속화:** SQLite. 복기 + 크래시 복구.
- **정합성 원칙:** 익일 매도(8)는 `DailyPlan`을 맹신하지 않고 **KIS 실보유 잔고를 조회해 청산**. Plan은 "의도", 잔고는 "실제".

### 컴포넌트 분리 원칙
- 분석·실행은 `DailyPlan` 하나로만 통신 → 독립 이해/테스트.
- `RiskGovernor`는 실행과 분리된 게이트 → 선정이 뭘 하든 한도 강제.
- **데이터 소스 어댑터**가 KRX(pykrx)/KIS에 묶이는 유일한 곳 → 백테스트/라이브 교체 시 어댑터만 교체.

## 4. 데이터 흐름 (하루 사이클)

```
══ 장 마감 전 (≈14:00) ══
 ① MarketRegime: 코스피/코스닥 지수·거래대금 → 시황등급 + 비중배수
 ② UniverseScreener: 대형주 모집단에 정량 팩터 → 후보 N개(예 30~50) 랭크
 ③ CatalystResearch: 후보별 뉴스 + 웹검색 → 재료/테마 구조화
 ④ Selector(LLM): 시황+후보+재료 → 바스켓 K개 + 확신도 + 갭다운 시나리오
 ⑤ RiskGovernor: 비중·한도·호가수용·블랙리스트 심사 → 승인 바스켓
 ⑥ CloseExecutor: 종가 단일가(15:20~15:30)에 매수 → 실체결 DailyPlan 갱신
 ⑦ ReportWriter: 전 과정 리포트 기록(MD)

══ 익일 시가 (≈09:00) ══
 ⑧ OpenExecutor: KIS 실잔고 조회 → 시초가 전량매도 → 실현손익 기록 → 복기
```

### 3가지 핵심 규칙
1. **방아쇠는 항상 룰.** LLM은 ③④(재료조사·선정)만. 매수/매도 실행·한도는 전부 룰.
2. **출구는 무조건.** 익일 시초 전량매도는 손익·갭 무관(MVP). 갭다운 시나리오는 기록만.
3. **잔고가 진실.** 실행은 KIS 실잔고에서 동기화하고 시작 — 내부 상태 맹신 금지.

## 5. 종목 선정 상세

### 5.1 Universe (모집단) — 대장주·대형주 중심
- **포함:** 코스피+코스닥에서 ① 당일 거래대금 상위 K ∩ ② 시총 ≥ 하한 ∩ ③ 주당가격·유동성 조건.
- **제외(안전):** 거래정지·관리종목·정리매매.
- 컷은 오픈 파라미터(§12). 초기값 예: 거래대금 상위 100 + 시총 ≥ 5,000억.

### 5.2 Stage 2 — 스크리너 팩터 (전부 결정론적, 일봉 기반)
**데이터 누수 방지:** 모든 롤링 통계는 *오늘(t) 포함 과거 N일*만. **정상성:** 절대가격 금지, 비율·거리·로그수익률만.

**① 거래대금** — 절대값 + 시장 내 상대순위(0~1).
**② 수급** — 외국인·기관 순매수(당일 + 최근 5일 누적). 정규화 = 순매수금액 / 거래대금. (백테스트: pykrx 투자자별 / 라이브: KIS 투자자별 매매동향 TR)
**③ 차트 팩터:**

```
정배열  Spread_t      = (MA_short − MA_long) / MA_long           # >0이면 단기선 위, 클수록 강함
        Alignment     = 1.0  if MA5>MA20>MA60>MA120
                      = (충족 쌍 수 / 3) 로 연속 보간            # 완전 역배열 → 0
신고가  Proximity_N   = Price_t / max(Price_{t..t-N+1})          # =1.0이면 N일 신고가
        DaysSinceHigh = t − argmax(Price_{t..t-N+1})            # 0이면 오늘 신고가(박스권 돌파)
거래량  VolRatio      = mean(양봉일 거래량) / mean(음봉일 거래량) # >1이면 상승매집 흔적
지지    NearMA        = 현재가가 MA5/MA20 ±x% 이내 + 직전 반등   # 눌림목 proxy(초기 단순형, 후속 정교화)
```

**랭킹:** 하드필터(모집단·거래정지·최소 거래대금/수급) 통과분에 팩터 가중합 점수 → **상위 N개**를 LLM 단계로. 가중치·룩백 N은 오픈 파라미터.

### 5.3 Stage 3 — CatalystResearch (LLM + 도구)
후보별로 뉴스피드(네이버/연합·DART 공시) 수집 + **GPT-5.4 웹검색** → 구조화 산출:
`{ has_catalyst, summary, strength, recency, theme, sources[] }`. 시스템 프롬프트는 프롬프트 캐싱으로 비용 절감.

### 5.4 Stage 4 — Selector (LLM, 스키마 강제 구조화 출력)
- 입력: 시황등급 + 후보(팩터) + 재료조사.
- 출력: 랭크된 바스켓 `{ symbol, conviction(0~1), rationale, theme, gap_down_scenario }[]`.
- GPT-5.4가 만쥬식으로 거래대금·수급·차트·재료·테마를 종합해 최종 K개 선정 + 확신도. 진입 전 **갭다운 대응 시나리오**를 종목별로 기록(MVP는 기록만).

## 6. Sizing & 리스크 거버너

```
종목 비중 = 자본 × 시황배수 × conviction   (바스켓 내 정규화)
   cap by  종목당상한 · 총익스포저상한 · 호가수용cap
```

- **시황배수:** BULL 1.0 / NEUTRAL 0.5 / BEAR 0.2 / HALT 0. + 만쥬의 "파격 비중"을 위한 **강세확신 tier(예 STRONG_BULL 1.5)** 옵션 — 단 항상 호가수용cap에 종속.
- **호가수용 cap:** 슬리피지 없이 담을 최대 수량. 초기 proxy = 일거래대금의 x%, 실행 직전 실호가로 재확인. (만쥬 "호가창이 받아주는 최대 물량".)
- **거버너 거부권(리스크 늘리는 행동만 거부):**
  - HALT(시황 서킷브레이커) → 매수 0.
  - 종목당상한·총익스포저 초과 → 감액 또는 제외.
  - 블랙리스트(거래정지·최근 연속손실) → 제외.
- **매도는 막지 않음** — 익일 시초 청산은 항상 통과(가속).

## 7. 실행 & 정합성

### 7.1 Stage 6 — 종가 매수
- 기본값 = **종가 단일가(15:20~15:30 동시호가) 매수**. 주문구분·지정가 톨러런스는 오픈 파라미터. (대안: 연속매매 막판 시장가 — 보류.)
- 체결조회로 실수량 확인 → `DailyPlan.basket[].filled_qty/avg_price` 갱신.

### 7.2 Stage 8 — 시초 매도
- 익일 **KIS 실보유 잔고 조회 → 시초가 전량매도**(시가 동시호가/시가 시장가). 실현손익 기록.
- 갭다운 시나리오는 기록만, MVP는 무조건 청산.

### 7.3 에러 / 정합성 (실거래 필수)
| 상황 | 대응 |
|---|---|
| 부분/미체결 | 체결조회로 실수량 확인 → 익일 매도는 실잔고 기준 |
| 크래시·재시작 | 내부 상태 맹신 금지, KIS 실잔고 동기화로 시작 |
| 중복 주문 | 주문ID 멱등성 + 당일 1회 실행 가드 |
| 14:00 스케줄 누락(머신 꺼짐) | 그날 진입 skip(반쪽 진입 방지) |
| 09:00 스케줄 누락 | **강한 알림**(오버나잇 방치 위험) + 수동 청산 경로 |
| 사람 개입 | Kill switch — 수동 전량 청산·중단 |
| API 오류 | 재시도 + 실패 시 알림, 해당 종목만 skip(전체 중단 X) |

## 8. 백테스트 & 검증

### 8.1 두 층 분리
| 층 | 단계 | 백테스트 |
|---|---|---|
| 룰 | 1·2·5·6·8 | **일봉으로 결정론적 재현 가능** |
| LLM | 3·4 | 과거 뉴스 point-in-time 재현 어려움 + 비결정 → 백테스트에서 **끄고 룰 baseline**, 효과는 포워드로 측정 |

### 8.2 엔진
- 과거 일봉 + 수급 + 시총(pykrx)으로 매 거래일 `종가매수 → 익일시초매도` 시뮬.
- **메트릭:** 승률·평균갭수익·MDD·회전율, **net**(수수료 + 거래세 0.18~0.23% 차감), 평균 슬리피지.
- **체결모델:** 단일가라 슬리피지 작음 → 매수=종가, 매도=시초가 근사(보수적 슬리피지·세금 차감), 호가수용cap 초과분은 부분체결 가정.
- **누수 방지 강제:** 팩터는 t 종가까지만, 매수 t 종가, 매도 t+1 시초.

### 8.3 데이터 소스 (어댑터 추상화)
- **백테스트 = pykrx (KRX EOD):** 일봉 OHLCV, 투자자별 수급, 시가총액·상장주식수, 상장/상폐 목록(생존편향 제거). 무료. 비공식·레이트리밋 주의.
- **라이브 = KIS REST:** 일봉·순위·수급·현재가·주문·잔고.
- 공통 `MarketDataProvider` 인터페이스로 추상화 → 정밀/운영 시 KRX 공식 OPEN API·유료 벤더로 교체 가능.

### 8.4 정직성
백테스트 = "기계적 엣지 거르는 1차 필터". LLM 선정 품질은 **소액 실거래 포워드 + 복기**로 최종 검증.

## 11. 빌드 순서 (Phase)

- **Phase 1 — 데이터·팩터·백테스트:** pykrx 데이터 프로바이더 + 팩터 계산 + 룰 스크리너 + 시황등급 + 백테스트 엔진(룰 baseline). → **기계적 엣지 확인 게이트.**
- **Phase 2 — LLM 분석:** CatalystResearch + Selector + DailyPlan + ReportWriter(드라이런 검증).
- **Phase 3 — 실행·자동화:** `order`/`balance` + Close/OpenExecutor + RiskGovernor + 스케줄러 + 에러/정합성 + **소액 실거래 포워드.**
- **Phase 4 — 복기 루프·확장:** 복기 컨텍스트 축적 + 점진 증액. (시가배팅은 별도 후속 모듈.)

## 12. 오픈 이슈 (단계별 확정)

- Universe 컷: 거래대금 상위 K, 시총 하한, 가격/유동성.
- 팩터 가중치·룩백 N: MA 기간(5/20/60/120), 신고가 윈도우, VolRatio 윈도우.
- 시황등급 임계치: 지수·거래대금 경계(만쥬 코스닥 12~13조/7~8조/5조 참고).
- 시황배수 tier 값, 종목당상한·총익스포저·호가수용cap %.
- 후보 N개수, 최종 바스켓 K개수.
- 종가 주문 방식(동시호가 지정가 톨러런스 vs 시장가), 시초 매도 방식.
- 뉴스피드 소스 확정(네이버/연합/DART) + 수집 방식, 웹검색 도구 구성.
- 수급 데이터: pykrx/KIS 필드 매핑.
- 스케줄러 플랫폼(mac launchd vs win), 알림 채널(텔레그램?).
