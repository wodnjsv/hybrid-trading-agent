# 재료 LLM 포워드 페이퍼 — 사전등록 (2026-06-05, 데이터 누적 전 동결)

> 데이터 누적 *전*에 합격선·방법·파라미터를 못박는다(p-hacking·optional stopping 방어). 동결 후 사후 변경 금지. 스펙 §5·§9, 적대리뷰 1·2차 반영.

## 1. 1차 추정량 (단일 — 다중성 차단, 적대리뷰 #4)
- **1차 = within-market paired 상대게이트**: LLM 매매일 한정, 그날 baseline 동수 `K_t`, day-level (LLM 평균 − baseline 평균) net의 평균 > 0.
- 다중성: KOSPI·KOSDAQ 2검정 → **사전지정 단일 추정량 = 시장구성비 고정 pooled**(두 시장 day-level diff를 한 표본으로 결합, 시장 더미 보정). 시장별 결과는 보조(탐색적)로만.
- band: 1차 = `net_s05`(편도 0.05% 슬리피지, 메가캡 현실). `net_s0`/`net_s10`은 민감도 보조.

## 2. 검정력 → 필요 N (적대리뷰 #3 — 정직한 기간)
- 검정: paired t (day 단위). **유효 N = 거래일수**(체결수 아님 — day 내 종목 상관).
- 가정: day-level diff SD ≈ 1.0~1.5%, 탐지 효과 = +0.2%/일(비용 허들 근방), 80% power, α=0.05 양측.
- 역산: N ≈ (2.8 × SD/효과)² ⇒ **~200~450 거래일(약 1~2년).**
- → **1차 판정은 N≈300 거래일 도달 후 1회**(보수 중앙값). 그 전 수개월은 **탐색적 모니터링 전용**(주간 `report`), **중단/연장 판정 금지(고정-N, 적대리뷰 #5).** N 미달 "유의X"는 *무결론*으로 보고(엣지 없음 아님).
- α = 0.05 (양측). 조기중단 없음.

## 3. 절대게이트 (배포-필수, 보조 아님)
- LLM net 평균 > 0 (보수 슬리피지 `net_s10`에서도). **실제 자본 투입의 문.**
- 단 진입 드리프트·갭다운 슬리피지에 편향될 수 있음(§6) → 과학적 1차 판정과 분리 보고.

## 4. 시점 누수 필터 (적대리뷰 #1)
- 재료는 **종가 동시호가 이전(≤15:20) 공개 정보만 인정.** GPT 시스템프롬프트가 강제 + `catalyst_timestamp` 로깅.
- 분석 시 `catalyst_timestamp > 15:20`인 픽은 **사후 제외**(누수 방지). 제외 비율도 보고.

## 5. 미시구조 매칭 (적대리뷰 #2 — 보고 항목)
- 매 리포트에 LLM vs baseline 바스켓의 **거래대금·vol20 중앙값 + 겹침도(Jaccard)** 동반 보고. 차이가 크면 결과를 매칭 조건부로 해석.

## 6. 동결 파라미터 (구현 상수와 일치)
- **모델 핀: `gpt-5`** (alias `gpt-5-2025-08-07`). *("GPT-5.4"는 존재하지 않아 `gpt-5`로 정정.)* 버전 변경 시 표본 절단.
- **매도세 `SELL_TAX = 0.0020`** (2026 현행 ~0.20% 가정; **운영 개시 전 현행 실세율 최종 확인·동결**). 수수료 편도 0.014%. 슬리피지 밴드 {0, 0.05%, 0.1%}.
- **baseline 정의:** 후보 shortlist(시장별 거래대금 상위 `SHORTLIST_N=30` ∩ 수급(t-1 외/기 둘다>0) ∩ 추세(close>MA20) ∩ 끝물회피(0<당일등락<10%)) 중 **거래대금 상위 `K_t`**(LLM 매매일) / 패스일은 `K_DEFAULT=3`.
- **LLM 후보풀 = baseline 풀 동일(MVP):** A/B 공정성 우선. 스크린이 신선 재료주를 거르는 over-filter 긴장은 **운영 후 재검토**(넓히면 페어링 재설계).
- 진입 = 확정 `close[d]`, 청산 = `open[d+1]`(1차). 다중호라이즌(`close[d+1]`/`close[d+2]`)은 보조 기록(병목 사후 분리).

## 7. 운영
- 매 거래일 **마감 후 `jongga-forward-eve`**(오늘 선별·기록), **익일(d+1) 마감 후 `jongga-forward-morn`**(전일분 정산 — settle은 d+1 EOD 바의 `open[d+1]`을 읽으므로 d+1 *마감 후* 실행; 매 저녁 eve와 함께 전일분 morn 권장), **주간 `jongga-forward-report`**(모니터링 전용).
- N≈300 거래일 도달 시 1차 판정 1회.
